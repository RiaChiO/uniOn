지금까지 나눈 패키지 설치, API 키 발급, 그리고 VS Code에서 .env 파일을 만들어 적용하는 전체 과정을 하나의 완벽한 가이드로 일목요연하게 합쳐 정리했습니다.

이 순서대로만 위에서부터 차근차근 진행하시면 세팅이 완료됩니다!

🚀 Gemini API 연동 및 환경 변수(.env) 세팅 전체 가이드
1. 필수 패키지 설치
백엔드 터미널(경로가 .../backend인지 확인)에 아래 명령어를 입력하여 필요한 라이브러리 2개를 설치합니다.

Bash
npm install @google/genai dotenv
@google/genai: 구글에서 제공하는 최신 공식 Gemini SDK입니다.

dotenv: API 키 같은 민감한 정보를 환경 변수(.env) 파일에서 안전하게 읽어오는 라이브러리입니다.

2. Gemini API 키 발급받기
웹 브라우저를 열고 Google AI Studio에 접속하여 구글 계정으로 로그인합니다.

화면 좌측 상단 또는 메인 화면에 있는 "Get API key" 버튼을 클릭합니다.

"Create API key" 버튼을 누르고 팝업창에 생성된 키를 즉시 복사하여 메모장 등에 따로 저장합니다.

⚠️ 주의: API 키는 생성된 직후 딱 한 번만 보여지므로 반드시 즉시 복사해야 합니다. 외부(GitHub 등)에 노출되지 않도록 절대 주의하세요.

3. VS Code에서 .env 파일 만들고 등록하기
🔗 1단계: .env 파일 생성
VS Code에서 현재 작업 중인 backend 폴더를 열어줍니다.

왼쪽 파일 탐색기 빈 공간을 마우스 우클릭한 뒤 [새 파일 (New File)]을 누릅니다.

파일 이름에 점(.)을 포함하여 .env 라고 정확히 입력하고 엔터를 칩니다.

⚠️ 주의: 확장자가 아니라 파일 이름 자체가 .env입니다. 뒤에 .txt 같은 것이 붙지 않도록 주의하세요.

🔗 2단계: 파일 내용 작성하기
새로 만든 .env 파일을 열고, 아래 형식을 복사해서 붙여넣습니다. AIzaSy... 부분에는 방금 구글 AI 스튜디오에서 복사한 본인의 실제 API 키를 넣어줍니다.

코드 스니펫
# Gemini API Key 세팅
GEMINI_API_KEY=AIzaSy구글에서_복사한_실제_키값
🚨 작성 시 핵심 주의사항 (안 지키면 서버가 뻗습니다!)

키값 앞뒤로 **공백(스페이스바)**이 들어가지 않게 붙여넣으세요.

값을 큰따옴표(")나 작은따옴표(')로 절대 감싸지 마세요. (예: GEMINI_API_KEY="AIzaSy..." ❌ ➡️ 따옴표 없이 생글씨로 작성해야 합니다.)

🔗 3단계: 파일 저장 및 올바른 위치 확인
내용을 다 적었다면 Ctrl + S를 눌러 파일을 저장합니다.

이 파일은 반드시 package.json 파일이 존재하는 backend 폴더의 최상위 루트에 있어야 합니다. (server나 routes 폴더 안으로 들어가면 백엔드가 환경 변수를 읽지 못합니다.)

4. 최종 확인 및 서버 재구동
세팅이 끝났다면 터미널 창에서 구동 중이던 서버를 Ctrl + C로 완전히 종료한 후, 아래 명령어로 백엔드 서버를 다시 켭니다.

Bash
npm run api:test
터미널 콘솔에 🔑 [Gemini Auth Check] API KEY 로드 상태: 정상 세팅됨 ✅ 메시지가 뜨면 모든 연동이 성공한 것입니다. 이제 프론트엔드 화면으로 가서 AI 기반 소모임 문맥 추천 기능을 마음껏 테스트해 보세요!
